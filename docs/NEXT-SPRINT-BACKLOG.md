# 下一阶段冲刺 Backlog（客服智能知识助手）

更新时间：2026-03-06

## 1. 冲刺目标
本轮冲刺目标不是继续扩功能，而是先完成产品收口、主链路稳态和测试基线建设。

本轮完成后，项目应满足以下结果：
1. 主界面收敛为客服知识助手，不再暴露旧的 PPT/方案生成主路径。
2. 上传、问答、SOP 三条核心链路具备基本稳定性。
3. 会话隔离从“浏览器本地记录”升级为“前后端端到端链路”。
4. 建立最小自动化测试基线，支持后续持续改造。

## 2. 冲刺范围
建议按 2 周冲刺执行。

优先级定义：
- P0：必须完成，否则不建议进入业务验收。
- P1：强烈建议完成，直接影响交付质量。
- P2：可作为冲刺尾部项，或顺延下轮。

负责角色建议：
- FE：前端
- BE：后端
- QA：测试
- PM：产品/业务

## 3. 任务清单

### P0-01 产品入口收口
- 任务：隐藏或下线前端 `generator`、`editor`、`draft` 相关入口，仅保留知识库、上传、SOP、客服问答相关导航。
- 负责人建议：FE
- 预估：1 天
- 依赖：无
- 验收标准：
  1. 侧边栏不再暴露旧方案生成入口。
  2. 首页文案不再出现 PPT、方案生成等旧目标描述。
  3. 用户主路径聚焦客服知识问答、上传管理、SOP 维护。

### P0-02 后端遗留能力隔离
- 任务：将 `draft/PPT` 相关接口标记为 legacy，并从主流程说明文档中移除。
- 负责人建议：BE
- 预估：1 天
- 依赖：P0-01
- 验收标准：
  1. README 和主计划文档不再将旧能力描述为当前主功能。
  2. 后端 legacy 路由有明确注释或分组。
  3. 新功能开发不再依赖 legacy 数据结构。

### P0-03 文案与乱码清理
- 任务：清理前端组件、默认数据、文档中的乱码内容，统一编码为 UTF-8。
- 负责人建议：FE + BE
- 预估：2 天
- 依赖：无
- 验收标准：
  1. 前端界面不存在明显乱码。
  2. 默认数据中的核心分类、标题、描述可读。
  3. 客服场景文案统一，不混入旧行业售前方案术语。

### P0-04 会话隔离链路
- 任务：新增并持久化 `sessionId`，聊天接口显式携带 `sessionId`，后端按会话存储历史或至少完成隔离透传。
- 负责人建议：FE + BE
- 预估：2 天
- 依赖：P0-01
- 验收标准：
  1. 浏览器首次进入生成并持久化 `sessionId`。
  2. `/api/chat` 与 `/api/solutions/:id/chat` 请求均携带 `sessionId`。
  3. 不同 `sessionId` 的对话不串扰。
  4. 至少具备基础会话元数据存储能力。

### P0-05 上传任务状态机
- 任务：为上传流程增加 `queued / processing / success / failed` 状态，并返回统一结构。
- 负责人建议：BE
- 预估：2 天
- 依赖：无
- 验收标准：
  1. 上传接口响应中包含状态字段。
  2. 失败任务可识别失败原因。
  3. 前端可展示处理中、成功、失败三种状态。
  4. 后续可扩展重试，不需要重写数据结构。

### P0-06 错误码与日志统一
- 任务：统一上传、聊天、SOP CRUD 的错误返回结构和服务端日志字段。
- 负责人建议：BE
- 预估：1.5 天
- 依赖：P0-04, P0-05
- 验收标准：
  1. 接口错误结构统一为可解析格式。
  2. 日志至少包含时间、接口、操作类型、结果、错误原因。
  3. 关键失败场景可通过日志定位。

### P1-01 客服结构化输出模板
- 任务：统一客服问答输出格式，至少包含推荐回复、处理步骤、核验项、升级建议。
- 负责人建议：PM + BE
- 预估：1.5 天
- 依赖：P0-04, P0-06
- 验收标准：
  1. 退款、投诉、账号、激活、升级五类问题的返回结构一致。
  2. 无足够引用时返回安全降级话术。
  3. 高风险场景包含人工复核提示。

### P1-02 SOP 命中展示优化
- 任务：将 SOP 命中结果和知识引用在前端清晰展示，支持客服快速判断可执行性。
- 负责人建议：FE + BE
- 预估：1.5 天
- 依赖：P1-01
- 验收标准：
  1. 回答中能区分“知识引用”和“SOP 建议”。
  2. 前端展示命中数量、来源标题和基础说明。
  3. 客服可以直接复制建议口径。

### P1-03 上传失败重试
- 任务：为上传失败任务增加手动重试能力。
- 负责人建议：FE + BE
- 预估：1 天
- 依赖：P0-05
- 验收标准：
  1. 失败任务可在界面上触发重试。
  2. 重试后状态重新进入 `queued/processing`。
  3. 相同任务不会生成重复脏数据。

### P1-04 基础审计日志
- 任务：记录上传、删除、SOP 新增/编辑/删除、问答调用的审计数据。
- 负责人建议：BE
- 预估：2 天
- 依赖：P0-06
- 验收标准：
  1. 至少可记录操作类型、对象、时间、结果。
  2. 问答请求可追溯到会话和主要输入摘要。
  3. 审计结构后续可用于看板统计。

### P1-05 数据样本清理
- 任务：清理旧的售前方案样本、无关能力数据和不适合客服场景的默认内容。
- 负责人建议：PM + BE
- 预估：1 天
- 依赖：P0-03
- 验收标准：
  1. 默认能力库聚焦客服业务。
  2. 默认草稿数据不再作为主路径依赖。
  3. 验收环境中看不到明显偏题样本。

### P1-06 客服五大场景验收集
- 任务：建立退款、投诉、账号、激活、升级转人工五大场景的标准问题集和期望输出。
- 负责人建议：PM + QA
- 预估：1 天
- 依赖：P1-01
- 验收标准：
  1. 每类场景至少 5 条问题样本。
  2. 每条样本有期望的结构化输出要求。
  3. 可直接用于回归和业务验收。

### P2-01 基础运营指标雏形
- 任务：输出最小看板数据接口，先覆盖问题量、未命中问题、SOP 命中率。
- 负责人建议：BE
- 预估：1.5 天
- 依赖：P1-04
- 验收标准：
  1. 可按天聚合问题量。
  2. 可统计无引用或低置信回答数量。
  3. 可统计 SOP 命中次数。

### P2-02 页面级状态提示优化
- 任务：补齐上传、聊天、失败重试、引用不足等状态提示。
- 负责人建议：FE
- 预估：1 天
- 依赖：P0-05, P0-06
- 验收标准：
  1. 用户能明确知道请求处理中、成功或失败。
  2. 错误提示不暴露内部异常细节。
  3. 引用不足时有明确人工复核提示。

## 4. 测试 Backlog

### T0-01 建立前端单元测试基线
- 任务：为 `storage`、聊天消息处理、上传表单校验建立首批单元测试。
- 负责人建议：FE + QA
- 预估：1.5 天
- 验收标准：
  1. 新增测试框架和基础运行脚本。
  2. 至少覆盖 3 个核心模块。
  3. 可在本地一键执行。

### T0-02 建立后端接口测试基线
- 任务：为上传、聊天、SOP CRUD 建立基础接口测试。
- 负责人建议：BE + QA
- 预估：2 天
- 验收标准：
  1. 覆盖成功和失败两类场景。
  2. 能 mock FastGPT 外部依赖。
  3. 关键接口回归可自动执行。

### T0-03 会话隔离测试
- 任务：验证不同 `sessionId` 的对话不会串话。
- 负责人建议：QA
- 预估：0.5 天
- 验收标准：
  1. 两组会话输入得到独立上下文结果。
  2. 历史记录读取不互相污染。

### T0-04 上传状态机测试
- 任务：验证上传从排队到完成的状态流转，以及失败重试逻辑。
- 负责人建议：QA
- 预估：0.5 天
- 验收标准：
  1. 状态流转可重复验证。
  2. 失败后重试可恢复。
  3. 重试不会重复插入脏记录。

### T1-01 五大场景业务回归
- 任务：基于客服五大场景进行冒烟回归。
- 负责人建议：QA + PM
- 预估：1 天
- 验收标准：
  1. 每类场景至少执行 5 条问题。
  2. 返回中包含引用或明确降级说明。
  3. 高风险类问题包含升级或复核提示。

### T1-02 打包与部署验证
- 任务：验证前端构建、后端启动、联调配置和最小发布流程。
- 负责人建议：FE + BE + QA
- 预估：0.5 天
- 验收标准：
  1. 前端可成功构建。
  2. 后端可正常启动并连通 FastGPT。
  3. README 中的启动步骤可复现。

## 6. 冲刺退出标准
满足以下条件才建议结束本轮冲刺：
1. 主路径已聚焦客服知识助手。
2. 会话隔离已端到端打通。
3. 上传具备可观测状态和失败重试能力。
4. 至少具备一套可运行的单元测试和接口测试。
5. 客服五大场景完成一轮业务回归。

## 7. 当前建议
如果只能先做 3 件事，优先顺序如下：
1. 产品入口收口 + legacy 隔离
2. 会话隔离 + 上传状态机
3. 测试基线 + 五大场景验收集
