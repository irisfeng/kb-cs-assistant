# 客服智能知识助手系统改造方案（2026-03-06）

## 1. 改造目标
将当前项目收敛为“客服智能知识助手系统”，围绕客服业务提供：知识检索、SOP 命中、结构化回复建议、升级决策与运营闭环。

## 2. 范围约束
1. 本轮不包含 PPT 生成功能。
2. 优先基于 FastGPT 社区版能力实现 MVP，不依赖商业版专属能力。
3. 上传支持两种入口：
   - 后台批量导入（主入口）
   - 前端管理页上传（后端中转）

## 3. 目标用户与场景
1. 一线客服：快速找到可执行回复和操作步骤。
2. 质检/运营：维护 SOP、复盘命中率与知识缺口。
3. 主管/二线：处理升级案例和高风险场景。

核心场景：退款退换、投诉安抚、账号权限、激活配置、升级转人工。

## 4. 现状与差距
### 4.1 已有能力
1. 文档上传与解析入库。
2. 全局问答与文档问答。
3. SOP/能力库 CRUD。
4. AI 草稿生成与编辑基础。

### 4.2 关键差距
1. 会话隔离和多用户边界不足。
2. 审计和运营指标缺失。
3. 客服域路由与结构化输出能力不足。
4. 文档体系存在历史目标残留，需清理统一。

## 5. 目标架构
1. 接入层：客服工作台（问答、上传、规则、看板）。
2. 业务层：
   - 知识接入服务（上传、解析、标签、入库）
   - 对话编排服务（意图识别、检索、回复生成）
   - SOP 引擎（命中条件、升级阈值）
   - 审计与分析服务（日志、指标、复盘）
3. 数据层：文档元数据、SOP 规则、会话记录、审计日志。
4. AI 层：FastGPT 工作流与知识库。

## 6. 分阶段实施（8 周）
### Phase 0（第 1 周）：聚焦与清理
1. 清理与当前目标不一致的文档与计划。
2. 冻结 MVP 范围与验收标准。
3. 建立客服五大场景测试集。

### Phase 1（第 2-3 周）：基础链路稳态
1. 完成上传双入口（后台批量 + 前端管理上传中转）。
2. 统一解析和入库流水线。
3. 实现会话隔离（sessionId）和聊天历史存储。
4. 修复乱码与统一文案资源。

### Phase 2（第 4-5 周）：客服 Copilot
1. 意图识别与场景路由。
2. 结构化输出模板：
   - 推荐回复
   - 处理步骤
   - 升级建议
   - 风险提示
3. SOP 规则命中与可解释引用。

### Phase 3（第 6 周）：版本与审计
1. SOP/模板版本历史。
2. 版本对比与回滚。
3. 操作审计日志。

### Phase 4（第 7 周）：运营看板
1. 高频问题 TopN。
2. SOP 命中率。
3. 转人工率、失败率、平均处理时长。
4. 知识缺口回流清单。

### Phase 5（第 8 周）：验收上线
1. 业务验收（真实客服场景）。
2. 技术验收（稳定性、性能、回归）。
3. 发布与回滚演练。

## 7. 交付物
1. 产品：场景说明、SOP 模板规范、验收清单。
2. 技术：接口文档、数据模型、部署与回滚手册。
3. 运营：知识治理流程、复盘模板、周报指标口径。

## 8. 验收标准（DoD）
1. 客服 5 大场景均可返回“引用 + 结构化建议”。
2. 上传流程稳定，失败任务可重试。
3. 会话隔离有效，无跨会话串扰。
4. SOP 命中与升级建议可追溯。
5. 运营看板指标可核对到日志。

## 9. 风险与应对
1. 知识质量不稳定：增加审核状态、过期提醒。
2. 模型回复偏差：低置信降级人工、模板约束。
3. 存储扩展性不足：从 JSON 逐步迁移到数据库。
4. 使用迁移阻力：提供样例库和上手手册。
