## 来源结论
1. 当前知识语料是“按产品拆分”的，不存在一套可直接覆盖所有 AI 产品的统一退款规则。
2. 已核验样本文档明确表明，退款/退订必须至少区分：
   - 产品名称
   - 订购渠道
   - 订购类型
   - 用户诉求是退订还是退费
3. 已核验样本文档明确写明：
   - 退订退费流程以省内为准
   - 退订不等于退费
   - 不同渠道、不同包型的规则不同
4. 因此，当用户只说“退增值业务”而未说明具体产品时，系统不应直接给出具体退费结论，而应先做分诊追问。

## 使用边界
1. 本 SOP 不是某个具体产品的退款规则，不可替代产品级 SOP。
2. 本 SOP 只回答“先核什么、先问什么、何时转人工”，不回答“某产品一定能退/不能退”。
3. 一旦用户补充了具体产品名，应优先切换到对应产品 SOP。

```md
# 天翼视联全局退款分诊 SOP v1

## 适用范围
适用于用户咨询“退增值业务”“能不能退款”“怎么退订”等，但尚未说明具体产品名称的场景。

## 核心原则
1. 未明确产品名称时，不能直接承诺退款或退订结果。
2. 退款与退订不是同一件事，必须先区分用户诉求。
3. 不同产品、不同订购渠道、不同订购类型的规则不同。
4. 具体业务订购渠道、资费政策、退订退费流程以省内及对应产品文档为准。

## 标准分诊顺序
1. 先问产品名称。
2. 再问订购渠道：营业厅/CRM、小翼管家、美好家旗舰店或其他。
3. 再问订购类型：自动续费月包、非自动续费服务包、一次性月/季/年包、连续包月。
4. 再问用户诉求：退订、退费还是售后。

## 标准回复口径
建议回复：请先提供具体产品名称、订购渠道和订购类型，不同增值业务的退订和退费规则不同，当前不能直接统一承诺退款结果。  
处理步骤：先确认产品名称、订购渠道、订购类型和用户诉求，再按对应产品规则判断。  
核验要点：产品名称、订购渠道、订购类型、支付方式、订单时间、退款原因、当前状态。  
升级建议：如用户信息不全、坚持要求明确退款结论，或问题已涉及故障、重复订购、未使用过等争议场景，建议人工复核。  
引用依据：当前知识库为按产品拆分管理，退款退订规则需按具体产品和渠道判断。

## 需要追问的信息
1. 产品名称
2. 订购渠道
3. 订购类型
4. 支付方式
5. 订单时间
6. 用户诉求是退订还是退费
7. 退款原因

## 禁止直接承诺
1. 不能在未明确产品名时直接回复“可以退款”。
2. 不能在未明确产品名时直接回复“不能退款”。
3. 不能把某个产品的退订规则套用到所有增值业务。
4. 不能忽略“以省内为准”的限制。
```
