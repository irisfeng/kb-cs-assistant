# 天翼视联物流异常：已签收但本人未收到 v1

## 适用范围
适用于用户反馈“物流显示已签收但本人没有收到”“签收状态与实际不符”“怀疑被误签收”等商城物流异常场景。

## 已核验依据
- 来源：`天翼视联商城物流与退换货 SOP v1` 的已核验商城客服文档结论
- 已核验原文要点：
  - 商品一般 `48 小时内安排发货`，具体以物流信息为准。
  - 当前已核验资料只明确了发货、物流、退换货和售后边界。
  - 当前资料没有给出“虚假签收、丢件、赔偿金额、固定补发时效”的标准流程。
  - 当用户咨询 `已签收未收到`、`物流一直不更新`、`要求赔偿` 等异常时，应先核商品、订单时间、当前物流状态和截图，再转商城售后人工复核。

## 使用边界
1. 当前已核验资料支持把“已签收未收到”识别为物流异常，而不是普通延迟。
2. 当前资料不支持承诺赔偿、补发、理赔时效、固定回访时限。
3. 因此这类问题应形成“先核验，再商城售后人工复核”的标准口径。

## 建议入库正文
```md
# 天翼视联物流异常：已签收但本人未收到 v1

## 适用范围
适用于用户反馈物流显示已签收但本人没有收到、签收状态与实际不符等场景。

## 当前已知边界
1. 当前已核验资料明确了商城物流、退换货和售后边界。
2. 当前资料没有给出虚假签收、丢件、赔偿金额、固定补发时效的统一规则。

## 标准回复口径
建议回复：您好，您反馈的“已签收但本人未收到”属于物流异常场景。请您先提供下单渠道、订单号或交易号、签收时间、物流截图以及是否有他人代收等信息，我们先协助核验当前物流状态，再提交商城售后人工复核。  
处理步骤：先确认下单渠道、订单号、签收时间、物流截图和代收情况；如确认物流状态显示已签收但用户实际未收到，按物流异常场景记录并转商城售后人工复核。  
核验要点：下单渠道、订单号或交易号、签收时间、物流截图、代收情况、联系人手机号。  
升级建议：如当前为已签收未收到、物流状态与实际不符，或用户要求赔偿、补发、明确时效，建议转商城售后人工复核。  
引用依据：当前已核验资料没有给出已签收未收到的固定赔偿或补发流程；此类异常应先核订单与物流信息，再转商城售后人工复核。  

## 禁止直接承诺
1. 不能直接承诺一定补发。
2. 不能直接承诺一定赔偿或给出固定赔偿金额。
3. 不能直接承诺固定处理时效。
```
