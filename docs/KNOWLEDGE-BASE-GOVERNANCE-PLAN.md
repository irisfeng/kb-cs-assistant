# 知识库治理方案

## 一、目标

本方案用于定义项目在生产环境中，如何对客服资料进行整理、分级、拆分、入库和使用。

本阶段重点解决 3 类常见生产问题：

1. 新旧版本同时命中，导致答案冲突
2. 内部运维或敏感信息混入普通客服回答
3. 不同产品资料混查，导致引用范围失控

## 二、推荐的生产结构

### 1. 全局规则库

用于存放所有产品共用的知识。

- 客服热线
- 升级转人工规则
- 退款、补偿原则
- 统一术语说明
- 标准回复模板

### 2. 产品客服库

按产品或稳定产品族建立独立知识库。

示例：

- 客流统计
- 车牌识别
- 区域入侵
- 视频 AI 算法舱

规则：

- 生产检索路径中只保留当前生效最新版
- 历史版本归档，不参与默认检索
- 用户问题命中产品后，优先查对应产品库

### 3. 设备与商城资料库

用于存放设备型号和商城商品资料。

- 摄像头型号说明
- 商城商品客服文档
- 型号兼容性说明

规则：

- 不与产品政策类文档混放
- 设备兼容信息优先按型号或 SKU 维度组织

### 4. 内部支持库

用于存放仅内部人员可见的资料。

- 默认账号密码
- 私网 IP 地址
- SSH 连接信息
- 部署维护步骤
- 排障脚本和内部操作说明

规则：

- 不向普通客服开放
- 不进入普通客服问答工作流
- 必须做角色权限控制

## 三、安全分级

每个文件入库前都必须先做安全分级。

- `PUBLIC_CS`：可供普通客服问答使用
- `INTERNAL_SUPPORT`：内部支撑知识，不面向普通客服
- `RESTRICTED`：含密钥、默认账号、私网地址、基础设施细节等敏感内容

如果同一份文档同时包含客服知识和敏感运维知识，必须先拆成两份再入库。

## 四、元数据要求

每个入库文档至少应维护以下字段：

- `source_dir`
- `file_name`
- `product_line`
- `product_name`
- `document_type`
- `channel`
- `region`
- `effective_date`
- `version`
- `version_rank`
- `is_latest`
- `security_level`
- `audience_scope`
- `import_scope`
- `status`
- `notes`

推荐取值：

- `audience_scope`：`CS_AGENT`、`INTERNAL_SUPPORT`、`RESTRICTED`
- `import_scope`：`GLOBAL`、`PRODUCT`、`DEVICE`、`INTERNAL`
- `status`：`CANDIDATE`、`APPROVED`、`ARCHIVED`、`BLOCKED`

## 五、入库规则

### 1. 版本治理

- 每个产品在生产检索路径中只能有一个生效版本
- 旧版本统一标记为归档
- 如无法判断哪个版本为最新版，则暂缓入库

### 2. 内容拆分

文档切块应按业务章节拆分，不能只按固定字数硬切。

适合单独成块的章节：

- 产品介绍
- 资费说明
- 订购规则
- 退订与退费规则
- 使用说明
- 设备兼容性
- FAQ

如果文档本身已有明确标题层级，不应把多个业务意图混在同一块中。

### 3. 敏感信息处理

入库前必须扫描以下内容：

- 私网 IP 地址
- 默认用户名和密码
- SSH 相关信息
- 管理后台地址
- “仅内部使用”“不对外提供”等表述

如果命中：

- 文档标记为 `RESTRICTED`
- 不进入普通客服知识库
- 先脱敏，或生成内部专用版本

## 六、检索与路由策略

生产环境中应优先采用“确定范围检索”，而不是“全库大范围召回”。

推荐流程：

1. 识别用户意图和产品
2. 先查询对应产品库或设备库
3. 必要时补查全局规则库
4. 合并结果后生成结构化回答
5. 如问题涉及内部实施信息，则升级到内部支持，不直接暴露敏感内容

## 七、FastGPT 映射建议

推荐的 FastGPT 应用结构：

1. 一个全局客服应用
2. 一个产品客服应用模板
3. 一个内部支持应用

推荐的数据集结构：

1. 一个全局规则数据集
2. 每个产品或产品族一个数据集
3. 设备问题较多时，每个设备族一个数据集
4. 一个或多个受限内部数据集

不要依赖 prompt 文字限制去实现单文档隔离或敏感信息隔离。

## 八、推荐操作流程

1. 扫描待入库目录，生成文档清单
2. 从文件名中提取产品、版本、日期、渠道等信息
3. 从正文中识别敏感信息
4. 按产品键识别最新版
5. 人工复核不明确或被拦截的文件
6. 对混合范围文档做拆分或脱敏
7. 将通过审核的文件导入目标数据集
8. 记录导入结果和 FastGPT 的 `collectionId`

## 九、在本仓库中的使用方式

当前仓库已提供一个目录扫描脚本，可用于生成：

- JSON 清单
- CSV 清单
- 重复版本组
- 敏感文件候选列表

后续所有批量入库动作，都应以该清单为基线，而不是直接对源目录执行导入。
